Model Crane

Small controlled crane made using Arduino

The model crane is a project developed using an Arduino board to control the movement of a small crane. It includes the use of servos for different axis movements. This project was built to explore concepts of embedded control and mechanical movement. I designed and assembled both the physical and electrical components, integrating software logic to allow for smooth operation.

Outline:

For my Discover Project I decided to program an Arduino to create a controller, skeleton, and hardware for a model crane. I built the skeleton of the crane out of cardboard, using half of a box as the base. On top of the base, I built a few extremities to make it look more like a crane, including a control room and a mast so that it had more reach. In terms of electronic components, I used an Arduino Minima R4 Microcontroller, 2 servo motors, 4 buttons, 4 330 Ω resistors, and wires to connect all of the components.

Design:

I wanted to create a small electronic machine that I could control to complete a small goal. I eventually landed on the idea of a crane just walking outside North Avenue, seeing the 500-foot machine erecting a new building right next to the street. I thought back to my childhood when I played with Cranky the Crane and my other Thomas the Tank Engine Trains. I thought back to the pure joy I had playing with that crane and being so excited when I would see a real one driving back from the grocery store with my mom. I decided to follow a simple Arduino Crane tutorial video with a few adjustments.

Problems:

I got all the electronic components in addition to a joystick component. I wrote code in the Arduino IDE to implement the joystick as the controller of the crane. The idea was that pushing the joystick left and right would control a servo at the bottom and pushing the joystick up and down would control a servo on top. The bottom servo controls the top servo’s position, and the top servo controls the arm of the crane. The crane arm got a wide range of motion from this simple two-servo method. Unfortunately, I did not get the correct wires to implement the joystick as a controller. It needed male-to-female wires, but I only had male-to-male wires to connect the components. I decided to pivot to use buttons as the arm controller. I used a method that was intuitive to me to indicate which buttons controlled which: XBOX button colors. I grew up playing on XBOX controllers, so in my mind yellow (Y) is up, green (A) is down, blue (X) is left, and red (B) is right. Although I didn’t put them in a D-Pad structure to give the user an intuitive controller, I was able to control the crane while rarely getting confused.

Hardware Implementation:

I used the Arduino’s VCC and GND probes to power the system and connected the servos and buttons to each of those lines. The buttons required a push up resistor because they didn’t need all 5 volts to function. Because I was using buttons, I had to write new code to implement the buttons as the controls. One thing about buttons is that you have to account for switch bounce. To take care of switch bounce, I added a 8 ms delay in the button detection loops so that it waited after detecting a change in the button status to actually read the new value. I did not implement a method that accounted for switch bounce and then read a continues button value, so in the current implementation, the crane arm is a bit jolty. Overall, I had a lot of fun learning about implementing this design with the microcontroller and electrical components. I also liked designing the physical crane structure out of cardboard so that it looked more crane-like!
